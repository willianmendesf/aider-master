# Regras
1. Funções devem possuir responsabilidade única.
2. Nomes devem ser autoexplicativos.
3. Evitar comentários redundantes.
4. Nunca ignorar erros.
5. Toda alteração deve manter legibilidade.
6. Código duplicado deve ser removido.
7. Dependências desnecessárias devem ser evitadas.
8. Preferir simplicidade à complexidade.
9. Sempre considerar manutenibilidade.
